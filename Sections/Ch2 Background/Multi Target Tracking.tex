Multi-target tracking (MTT) converts a sequence of noisy radar detections into stable estimates of target state over time. Each radar update produces a set of detections characterised by range, bearing, and elevation, as described in the preceding sections. The role of the tracker is to associate these detections across successive updates, filter out noise, and maintain a continuous estimate of each target's position and velocity to support detect-and-avoid (DAA) decision making.

A tracking filter combines a model of target motion with incoming measurements to produce an optimal state estimate. The Kalman filter \cite{jazwinski1970stochastic} is the standard framework, alternating between a prediction stage, in which the current state is propagated forward using a motion model, and an update stage, in which a new measurement corrects the prediction. The constant velocity (CV) model is the most common motion model, with process noise absorbing small deviations from straight-line flight. For manoeuvring targets the interacting multiple model (IMM) \cite{blom1988imm} maintains a bank of filters under different motion assumptions and blends their outputs probabilistically.

Since radar measurements are expressed in spherical coordinates while motion models are most naturally formulated in Cartesian space, the measurement equation is nonlinear. The Extended Kalman Filter (EKF) handles this via first-order linearisation, while the Unscented Kalman Filter (UKF) \cite{julier1997newextension} propagates sigma points through the nonlinear transformation for improved accuracy when the nonlinearity is significant.

At each update the tracker must determine which detections correspond to which tracks. Gating first restricts candidates for each track to detections falling within an ellipsoidal region around the predicted measurement, based on the Mahalanobis distance, reducing both computational cost and spurious assignments. The Global Nearest Neighbour (GNN) algorithm then solves the assignment across all tracks and detections simultaneously as an optimisation problem.

In dense clutter, hard assignment methods can be insufficient. Probabilistic data association (PDA) updates each track as a weighted combination of all gated detections, with weights proportional to measurement likelihood. Joint PDA (JPDA) extends this to the multi-target case. Multiple hypothesis tracking (MHT) defers the association decision by maintaining a tree of hypotheses across multiple scans (see Figure \ref{fig:mht}).

Detections that cannot be associated with any existing track initialise tentative tracks, which are promoted to confirmed only after receiving consistent detections across a minimum number of updates — commonly an M-of-N logic. This prevents transient clutter returns from generating false confirmed tracks. Conversely, tracks accumulating consecutive missed detections are allowed to coast for a limited number of updates before being deleted. The promotion and deletion thresholds balance responsiveness to new targets against robustness to clutter and intermittent detection loss.
