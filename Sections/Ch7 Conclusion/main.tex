CFAR was validated, with achieved false alarm rates closely matching desired values.

Implementation choice strongly affected runtime, latency, and throughput across hardware platforms.

Hardware context changed which implementation was most suitable, supporting hardware-aware pipeline design.

Contributed design guidance with relevance beyond the immediate industry project.
