\section{Testing Datasets}
Three types of datasets are used for the experiments outlined in the following sections (see Sections \ref{sec:processing_app}, \ref{sec:tracking_app}, and \ref{sec:orchestration}):
\begin{itemize}
    \item \textbf{Generic Sine Waves with Noise}: To validate the correctness of our processing algorithms as well as their speed, we generate binary files of synthetic radar data consisting of a complex sine wave with additive white Gaussian noise. This allows us to test the software under controlled conditions where we can precompute the expected output, and to evaluate the performance of the software in terms of latency and throughput without the confounding factors of real-world data. (INCLUDE FIGURE HERE illustrating the signal)  
    \item \textbf{Simulated Radar Data}: To evaluate the performance of our algorithms under more realistic conditions with a known ground truth, we use simulated radar data generated by the Ocean Acoustics library \cite{}. This library allows us to model the radar signal propagation, target reflections, and noise characteristics in a way that closely mimics real sonar and radar returns. By using this dataset, the accuracy of stages such as monopulse angle discrimination can be evaluated with realistic phases and amplitude differences between channels. Furthermore, this library allows us to simulate multiple targets and clutter, allowing for full integration tests and visualisations of the tracking performance in real time.     
    \item \textbf{Real Radar Data}: To validate the performance of our software in real-world conditions, we use data collected from AWR radar chips. This dataset includes both false target data, where the chip outputs a clean compressed pulse corresponding to an imaginary target, and real target data (where actual targets are present). Testing with this dataset allows us to evaluate the robustness of our algorithms to real-world (non-Gaussian) noise, unique target signatures, and realistic clutter, as well as, potential timing, data corruption and memory issues that may arise when running the software in real time.
\end{itemize}
